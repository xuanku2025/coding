# ZStack ZSphere Web 交互与设计规范 (Skill)

本 Skill 用于指导和生成与 ZStack ZSphere 虚拟化平台（及类云计算控制台后台）视觉一致、交互连贯的 Web 前端页面。如果你需要设计类似的云计算管理平台，应严格遵循本指南。

## 0. 官方界面术语与布局（ZStack 用户手册 v4.10.25）

以下为 ZStack 官方文档中的用户界面定义，设计时请与之一致：

| 官方名称       | 位置与作用 |
|----------------|------------|
| **导航栏**     | 界面左侧，主要功能入口：首页、资源清单、系统管理等。与资源树共同构成主导航框架，支持一二级菜单切换。 |
| **资源树**     | 资源清单二级菜单下方，树形结构展示虚拟机、数据存储、分布式交换机等；支持展开/折叠、右键菜单。 |
| **资源详情页** | 主内容区，展示当前选中资源的详细信息（配置总览、监控、关联资源），以卡片、列表等形式呈现。 |
| **标签页**     | 资源详情页上部，同一页面内切换不同功能视图（总览、监控、快照等），配合操作栏实现「选中-查看-操作」闭环。 |
| **操作栏**     | 资源详情页顶部或侧边，针对当前资源的操作按钮（开机、控制台、修改配置等）。 |
| **任务/报警窗口** | 界面底部，全局可见；展示平台操作任务与实时报警，支持展开/折叠。 |

设计目标（官方）：直观导航、智能关联视图，高效完成资源的创建、监控与运维。

## 1. 设计语言基础 (Foundation)

该平台是在 **Ant Design** 的基础上进行深浅色定制与信息密度重构的企业级中后台系统。

- **Primary Framework**: 默认优先使用 `Ant Design` 或兼容此生态的 UI 组件库。
- **UI 气质**: 高度专业、严谨、高信息密度。去除不必要的装饰性元素，聚焦数据本身。
- **图标系统 (Iconography)**: 使用直观的线性或面性图标，资源状态图标必须能够直接映射其实体状态。

## 2. Design Tokens (视觉变量)

在实现 CSS 变量（如 CSS/LESS/SCSS Variables）时，请使用以下标准映射：

### 颜色系统 (Color Palette)
- **品牌主色 (Primary)**: `#1890ff` (天空蓝，默认) - 用于主按钮、活跃标签、选中项、进度条等。
- **顶栏背景 (Header/Hero)**: `#004fbb` ~ `#001529` - 全局顶部栏或极其重要的侧边栏背景深色，彰显企业重工产品感。
- **页面背景 (Layout Bg)**: `#f0f2f5` - 在白色卡片外围做底托区分。
- **组件表面 (Surface)**: `#ffffff` - 卡片、面板、内容区域的核心背景。

### 状态色 (Semantic/Status Colors)
在虚拟化中，状态极其关键（如虚拟机的启停状态）:
- **Success / Running (运行中/已连接)**: `#52c41a`
- **Error / Stopped / Critical (关机/错误/紧急)**: `#ff4d4f`
- **Warning / Serious (警告/严重)**: `#faad14`
- **Info / Processing (提示/处理中)**: `#1890ff`
- **Disabled / Other (其他/离线)**: `#bfbfbf` 或 `#d9d9d9`

### 尺寸与排版 (Typography & Spacing)
- **字体族 (Font Family)**: `PingFang SC`, `Microsoft YaHei`, `Helvetica Neue`, `Arial`, sans-serif
- **字体基础**: 标准正文使用 **14px**。标题部分采用 **16px, 20px, 24px** 建立层级关系。
- **行高 (Line Height)**: 一般文本取 `1.5` ~ `1.5715`。信息密集表格内行内距可压缩。
- **空间栅格 (Spacing)**: 核心组件间内边距与外边距统一使用 `8px` 及其倍数（如 8px, 16px, 24px）。
- **圆角 (Border Radius)**: 偏向硬朗直接的视觉感，默认组件采用 **2px** (如输入框、Select)，外围卡片偶尔用 **4px**。

## 3. 核心交互组件规范 (Components)

### 3.1 登录页设计 (Login Dashboard)
- **布局**: 屏幕正中白底浅色卡框，极简背景。
- **设计重点**: 居中对齐，大面积留白。采用带图图标的输入框 (前缀为用户或密码图标)，主操作为大号全宽品牌色按钮。输入框聚焦时 (Focus)，采用清晰的 `#1890ff` 纯色边框强调，而非大范围的光晕扩展。

### 3.2 导航模块 (Navigation)
- **侧边栏 (Sidebar)**: 采用两级折叠层级设计。最左侧常驻仅图标（或带微小文字）的一级导航面板；点击后在其右侧贴合弹出一级导航详细内容的“TreeView (树状视图)”或二级菜单列表。
- **分组列表栏 (Grouped Menu)**: 在系统管理或监控栏目下，二级菜单有浅灰色的分组小标题（如"报警服务", "任务事件"），字号较小，字体偏暗，起视觉区隔作用。
- **顶部操作流**: 包含“全局搜索 (Global Search)”并用显著快捷键提示 (`Ctrl+K`)；侧边提供用户菜单、语言切换。
- **底部状态栏 (Footer Dock)**: 常驻一条用于展示“任务 (Tasks)”和“平台实时报警 (Alarms)”的抽屉区，配备红蓝色未读消息数字小红点 (Badge)。
- **面包屑导航 (Breadcrumb)**: 详情页顶部必备组件（如：数据中心 > 集群 > 主机 > 虚拟机），提供层级方位感。

### 3.3 Dashboard 与复合卡片 (Metrics & Cards)
- **复合操作带 (Action Bar)**: 在复杂的监控或报警面板上方，使用紧凑的工具栏聚合全部操作，顺序通常为：`[↺ 刷新] [确认/主操作] [批量下拉] [筛选 Select * N] [导出/下载图标]`。
- **辅助提示图标 (Tooltip Icons)**: 指标卡片或表单标签侧面经常配备浅灰色的圆形问号 `(?)` 或 `(i)` 图标 (`#bfbfbf`)，Hover 时展示黑色背景的文字 Tooltip 解释专业术语。
- **大数字展示**: 重要统计指标采用 **大数字 (24px - 36px) + 次要说明字** 的组合。
- **可视化图形 / 组件**:
  - **环状图 (Semi-circle Gauge)**: 展示资源使用率（CPU, 内存, 存储）。环形带颜色需按阈值切换（如 80% 以上变警告色）。
  - **进度条 (Progress Bar)**: 在面板明细中表现资源用量（如存储物理已用、可用）时，使用蓝色窄长条进度带标注当前容量及总容量占比。
  - **点阵状态**: 用（左侧原点，右侧数量/文案）表现分布规律。

### 3.4 列表与表格 (Data Tables)
- **表格列显示**: 表头通常带浅灰背景 `#fafafa`，加粗。
- **批量与筛选操作**: 表格顶部必须配备组合操作栏（如“新建”、“开机”、“批量操作”下拉）和**可折叠搜索栏**，并在表格最左侧预留 Checkbox 列以支持多选操作。
- **状态可视化**: 任何具有状态属性（如：主机连接状态、虚拟机电源状态）的字段，一律采用 **“颜色原点(Dot) + 文本”** 的渲染方式，不得仅使用枯燥文字。
- **行操作菜单 (Row Actions)**: 当单行数据存在多个操作动作时，将低频操作收纳在行末尾的一个带有 `···` (省略号图标) 的幽灵按钮内，点击悬浮出下拉菜单。
- **分页组件 (Pagination)**: 常驻右下角，配置总条数提示和快速跳转组件。

### 3.5 详情页结构 (Detail Pages)
采用典型的云计算资源属性面板设计 (Master-Detail)：
- **Hero Title & Icon Card**: 左上方显著展示资源名称，旁配有一个大号的正方形图标底板（蓝底白图标）来指示该资源类型。其下方是特定的黑灰色背景块强调离线状态或高亮指示块。
- **行内文字链 (Text Link Actions)**: 为了降低按钮的重量级感，通常会在某模块右上方使用 "带有小图标的文字链接" 作为轻量操作（如"接管现有存储"）。
- **上部操作组 (Action Group)**: 提供常用高频功能，如“开机”、“打控制台”、“修改配置”按钮。
- **选项卡切换 (Tabs)**: 下方用长 Tabs 切换核心模块（如：总览、监控、快照、备份、报警、高级设置）。
- **折叠描述面板 (Collapsible Descriptions)**: 采用标题头带下拉箭头，内容区域包含标签向左对齐/右对齐、带有浅灰色细边框的字段栅格 (`Label: Value`) 进行属性罗列。

### 3.6 弹框与表单 (Modals & Forms)
- **新建导向 / 分步设计**: 
  - 对于短表单，采用标准带标题、底部含【取消】【确定】按钮的居中 Modal。
  - 对于创建虚拟机、添加节点等复杂流程对象，应当使用**垂直拆分布局的 Modal**（左侧 Tabs 导航、右侧表单内容）或者直接使用 **Steps 分步向导**。
- **表单排版**: 标签 (Label) 与输入框呈水平对齐（左 25%，右 75%）结构，保证阅读流顺畅。
- **必填提示**: 使用红色星号 `*` 标识必须填写的项，校验提示紧贴输入框下沿以红色字样显示。

## 4. 落地建议 (Implementation)

代码生成时，为了贴近此 Skill，请直接引入 `Ant Design` 或基于其类名规范构建 CSS Modules：

```css
/* 示例: 全局根变量配置参考 */
:root {
  --primary-color: #1890ff;
  --success-color: #52c41a;
  --warning-color: #faad14;
  --error-color: #ff4d4f;
  --heading-color: rgba(0, 0, 0, 0.85);
  --text-color: rgba(0, 0, 0, 0.65);
  --text-color-secondary: rgba(0, 0, 0, 0.45);
  --disabled-color: rgba(0, 0, 0, 0.25);
  --border-radius-base: 2px;
  --border-color-base: #d9d9d9;
  --box-shadow-base: 0 3px 6px -4px rgba(0, 0, 0, 0.12), 0 6px 16px 0 rgba(0, 0, 0, 0.08), 0 9px 28px 8px rgba(0, 0, 0, 0.05);
  --font-family: 'PingFang SC', 'Microsoft YaHei', 'Helvetica Neue', Arial, sans-serif;
}
```

遵循以上原则将极大提高新建界面的业务一致性。

## 5. 如何基于实际平台补充本 Skill（你本地可访问时）

本 Skill 的 token 与组件描述基于 Ant Design 惯例与公开文档整理。若你能在内网访问实际 ZSphere 控制台（如 `https://<内网IP>/virtualization-dashboard`），建议按下面方式校准与扩充本 Skill，便于后续交互设计参考：

1. **提取 Design Tokens**
   - 浏览器打开平台 → F12 开发者工具 → 选中顶栏、按钮、卡片、表格等元素，在 **Computed / Styles** 中记录实际使用的颜色、字号、圆角、阴影（如 `--primary-color`、`border-radius`、`box-shadow`）。
   - 将实际数值补充到本文「2. Design Tokens」中，或新建 `tokens-from-live.md` 与本文对照使用。

2. **核对交互与布局**
   - 对照「0. 官方界面术语与布局」与「3. 核心交互组件规范」，确认导航栏、资源树、标签页、操作栏、任务/报警窗口的位置与交互（展开折叠、右键菜单、多选批量操作等）是否与现网一致，不一致处在本 Skill 中修正或备注版本差异。

3. **截图与关键流程**
   - 对登录页、首页/Dashboard、列表页、详情页（含标签与操作栏）、典型弹窗/分步向导各截 1～2 张图，保存到项目内（如 `docs/zsphere-screenshots/`），在本 Skill 或 README 中引用，便于后续「按图还原」或评审一致性。

4. **快捷键与可访问性**
   - 若平台有全局搜索等快捷键（如 Ctrl+K），在「3.2 导航模块」中补充；若有焦点顺序、ARIA 等可访问性约定，可单列小节记录。

完成上述补充后，在 Skill 顶部或会话中说明「已按实际平台校准」，后续设计即可优先以你整理后的 token 与交互为准。
