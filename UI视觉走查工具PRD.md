# UI 视觉走查工具 PRD

## 1. 产品概述

| 项目 | 内容 |
|------|------|
| 产品名称 | UI Inspector |
| 产品定位 | 帮助设计师快速走查前端 UI 视觉还原度，并一键提交 Bug 的效率工具 |
| 目标用户 | UI 设计师、前端开发、测试工程师 |

---

## 2. 核心流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  选择 Figma  │───>│  捕获页面   │───>│  视觉对比   │───>│  Jira 提交  │
│   设计稿    │    │   截图      │    │  标注差异   │    │    Bug      │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

---

## 3. 功能列表

### 3.1 Figma 设计稿获取
- [ ] Figma OAuth 授权登录
- [ ] 选择 Figma 文件/Page/Frame
- [ ] 获取设计稿截图
- [ ] 提取设计稿标注信息（可选）

### 3.2 前端页面捕获
- [ ] 输入页面 URL
- [ ] 选择视口尺寸（1920x1080 / 375x667 / 768x1024）
- [ ] Puppeteer/Playwright 截图
- [ ] 指定 DOM 元素截图（可选）

### 3.3 视觉对比
- [ ] 叠加模式（半透明覆盖）
- [ ] 滑块模式（左右拖动对比）
- [ ] 并排模式（左右排列）
- [ ] 差异高亮（自动标记像素差异）
- [ ] 缩放和平移操作

### 3.4 Bug 标注
- [ ] 点击标记差异点
- [ ] 选择问题类型：颜色 / 字体 / 间距 / 尺寸 / 其他
- [ ] 填写问题描述
- [ ] 自动截取差异区域截图
- [ ] 问题列表管理（增删改）

### 3.5 Jira 集成
- [ ] Jira OAuth 授权 / API Token 登录
- [ ] 选择 Jira 项目
- [ ] 选择 Issue Type（Bug）
- [ ] 自动填充：标题、描述、截图附件
- [ ] 支持自定义字段映射

### 3.6 报告导出
- [ ] 导出走查报告（PDF）
- [ ] 导出 Markdown 格式
- [ ] 走查历史记录

---

## 4. 页面结构

### 4.1 首页
- 快捷入口：开始走查、历史记录
- 最近走查列表

### 4.2 走查页面
- 左侧：Figma 设计稿
- 右侧：前端页面截图
- 中间：对比操作栏
- 底部：问题清单

### 4.3 设置页面
- Figma 授权配置
- Jira 授权配置
- 默认视口设置

---

## 5. 技术架构

| 模块 | 技术选型 |
|------|----------|
| 前端框架 | React + TypeScript |
| 截图引擎 | Puppeteer / Playwright |
| 图像对比 | Resemble.js |
| 状态管理 | Zustand |
| UI 组件 | Radix UI / Shadcn UI |
| 样式 | Tailwind CSS |
| 构建工具 | Vite |

---

## 6. Jira 提交字段映射

| 字段 | 来源 |
|------|------|
| Summary | `[UI走查] 页面名称 - 问题描述` |
| Description | 详细问题说明 + 差异截图 |
| Issue Type | Bug |
| Priority | 根据严重程度自动选择 |
| Labels | ui-inspector, visual-regression |
| Attachment | 差异截图图片 |

---

## 7. 非功能需求

- 页面加载时间 < 3s
- 截图生成时间 < 5s
- 支持 Chrome / Edge / Firefox
- 数据本地存储（可选云端同步）
