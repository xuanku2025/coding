# 07 后续开发与迭代计划 (V1.1 - V2.0)

在 V1.0 (MVP) 版本稳定上线并获得一批种子用户后，我们将根据用户反馈和数据表现，开启后续的迭代计划。本章规划了小程序在中长期的发展方向和功能演进。

## 1. V1.1 版本：体验打磨与基础社交能力增强 (预计发布后 1-2 个月)

V1.1 版本的核心目标是**解决 MVP 版本暴露的痛点**，提升记录的准确性和分享的多样性。

### 1.1 核心记录体验优化
*   **算法增强：**
    *   **引入卡尔曼滤波 (Kalman Filter)：** 更平滑地处理 GPS 漂移点，提升轨迹在城市高楼区和树林中的准确度。
    *   **海拔数据采集：** 调用更底层的气压计或高度 API（如果微信侧支持更好），增加总爬升/下降数据展示。
*   **防断点机制强化：**
    *   优化 `Storage` 缓存续传逻辑，确保即便手机没电自动关机，重启后仍能找回之前的运动数据。

### 1.2 分享与视觉体验升级
*   **更多海报模板：**
    *   增加 3-5 套不同风格的数据浮层水印（极简风、赛博朋克风、大字报风）。
    *   支持用户自定义数据块的位置和文字颜色。
*   **动态轨迹回放 (Canvas 动画)：**
    *   在结算页增加一个“轨迹回放”按钮，用动画的形式在地图上重新画一遍刚才走过的路段，用户可录制该动画发视频。

### 1.3 基础社交互动 (轻社交)
*   **点赞与排行榜 (微信原生排行榜)：**
    *   接入微信开放数据域 (Open Data)，实现基于微信好友的“本周跑量排行榜”和“单次最长距离排行榜”，纯粹刺激，不引入复杂评论系统。

---

## 2. V1.5 版本：设备互联与垂直场景深耕 (预计发布后 3-5 个月)

V1.5 版本的核心目标是**满足进阶用户的需求**，并尝试破圈获取不同运动场景的用户。

### 2.1 穿戴设备联动
*   **蓝牙心率带接入：**
    *   利用微信小程序的 Web Bluetooth API (`wx.createBLEConnection`)，支持直连标准的蓝牙心率带/心率臂带。
    *   在面板和结算页增加实时心率显示和心率区间统计图。
*   **Apple Watch / 智能手表独立版探讨：**
    *   调研是否可以开发简单的 WatchOS / WearOS 配套应用（这可能超出小程序的范畴，需评估成本），或只是在前端界面优化对各类手表的同步数据展示格式。

### 2.2 垂直运动场景扩展
*   **路线路书导入与导航 (针对骑行)：**
    *   支持上传 GPX/KML 路线文件。
    *   在地图上叠加显示目标路线，如果偏离路线则进行语音/震动提醒。
*   **间歇跑/自定义训练计划 (针对跑步)：**
    *   允许用户设定“跑 3 分钟，走 1 分钟，循环 5 次”的训练计划。
    *   并在到达阶段节点时触发强烈的语音/震动反馈。

## 3. V2.0 版本：社区化与商业化探索 (长期愿景)

V2.0 代表产品已经拥有了稳定的月活用户（MAU）和品牌认知。此时考虑适度的社区沉淀和自我造血能力，但**仍必须坚守“极简、不扰民”的初心**。

### 3.1 兴趣圈子与活动聚合
*   **轻量级俱乐部/车队：**
    *   用户可以创建基于微信群的“俱乐部”。
    *   俱乐部拥有专属的小程序落地页，展示总里程和内部成员排行榜。
*   **线上挑战赛：**
    *   例如“周末 10km 挑战”，完赛后可解锁专属的电子勋章或绝版海报贴纸。

### 3.2 极简商业化尝试 (绝不弹窗、绝打开屏)
*   **高级海报与数据包 (内购)：**
    *   默认提供 3 套海报，高级的动态 3D 轨迹海报、专业数据分析图表打包成“Pro 订阅”或单次低价解锁（如 1 元/次）。
*   **品牌联名水印/活动赞助：**
    *   与运动品牌合作，推出带有品牌 Logo 和 Slogan 的限定版分享海报边框，这是一种极具传播力的轻植入。

---
> 综上，本项目将始终围绕“记录质量”和“分享品质”这两个核心飞轮，克制做加法，用最轻量的形态服务热爱运动的用户。
