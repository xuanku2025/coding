# 01 竞品调研与分析

## 1. 调研目标

- **了解市场现状**：分析当前主流运动类App和微信小程序的功能、定位与用户群。
- **挖掘痛点与机会**：找出大厂没有做好的地方或小程序的独特优势场景（如轻量、社交属性）。
- **提炼核心功能**：确定哪些是必须有的（Must-have），哪些是差异化竞争点（Nice-to-have）。
- **借鉴优秀设计**：学习行业标准的UI交互（如大字号数据显示、便捷的开始/暂停操作）。

## 2. 竞品选择

| 竞品名称 | 类型 | 核心定位 | 主要参考点 |
| :--- | :--- | :--- | :--- |
| **Keep** | 独立 App | 全民运动社区，包含几乎所有运动类型 | UI设计规范、数据展示的丰富度、社区互动模式 |
| **悦跑圈** | 独立 App | 专注于跑步文化与赛事 | 跑步专属数据的专业度（如步频、步幅）、社交分享海报设计 |
| **咕咚** | 独立 App | 智能硬件+运动大数据 | 硬件连接能力、活动运营、轨迹算法准确度 |
| **Strava** | 独立 App (全球) | 骑行、跑步硬核玩家的“朋友圈” | 路线路段探索（Segments）、极简无干扰的记录体验、极强的心流设计 |
| **微信运动** | 微信内置功能 | 全民每日步数轻社交 | 极低门槛的参与感、基于微信好友的排行榜极具粘性 |
| **各品牌打卡小程序** | 微信小程序 | 营销/活动类（如品牌线上马拉松） | 小程序环境下的定位权限获取方式、轻量级打卡体验、裂变分享机制 |

## 3. 详细竞品分析维度

### 3.1 核心记录体验（轨迹与数据）
*   **App阵营（Keep/悦跑圈/Strava）：**
    *   **优势：** 轨迹精度极高，支持后台长期保活防杀；数据维度全面（配速、海拔、心率、步频等）；支持连接Apple Watch/Garmin等外设。
    *   **劣势：** 启动慢、广告多（除了会员版的Strava）、对于只想要“简单记录一次”的用户来说太重。
*   **小程序阵营（一般性运动小程序）：**
    *   **优势：** 即用即走，扫码或群内链接直接拉起；分享极其丝滑（直接发卡片到聊天窗口）。
    *   **劣势：** 容易被微信系统回收（切后台久了轨迹会断）；无法直接获取系统底层的丰富传感器数据；受限于微信性能，长轨迹渲染可能卡顿。

### 3.2 社交与分享机制
*   **Keep/咕咚：** 构建了封闭的App内社区，鼓励发动态。想要分享到微信，需要生成长图再跳转，路径较长。
*   **悦跑圈：** 最擅长做“跑步水印贴纸”和轨迹照片。水印设计极具传播力，跑者非常喜欢晒图。
*   **Strava：** 以地图轨迹+照片+硬核数据为主的分享风格。
*   **我们的机会点（微信小程序）：** **“原生社交裂变”**。我们可以做到：运动结束 -> 一键生成精美海报贴纸 -> 直接唤起微信好友/朋友圈分享窗口，路径最短。无需跳出微信。

### 3.3 商业化干扰度
*   **市场现状：** 几乎所有头部App在开屏、首页、弹窗都有大量无关广告、课程推销、硬件售卖。
*   **我们的机会点：** **“极致纯粹”**。主打无广告、秒开、一键开跑/骑行。

## 4. 痛点总结与差异化定位

### 用户痛点分析：
1. **App太臃肿：** 更新频繁，每次打开要看5秒广告，想马上开始跑步却被一堆弹窗阻挡。
2. **分享门槛高：** 辛辛苦苦跑完，App分享到朋友圈经常只是一条干巴巴的链接或者画质被压缩的图片。
3. **不想社交绑架：** 很多用户只想自己默默记录，或者只发给特定两三个跑友群，不想混迹于陌生的App社区。
4. **设备没电焦虑：** 复杂的App耗电量极大。

### 我们的产品差异化定位：
*   **Slogan：** 纯粹记录，即刻出发。
*   **定位：** 一款**无广告、秒启动、专注于轨迹记录与微信内极速分享**的轻量级跑步/骑行工具。
*   **核心打法：**
    *   **极简：** 首页只有巨大的地图和一个“开始”按钮。
    *   **好看：** 投入大量精力在**结束后的分享海报生成**上（提供地图盲盒叠加、动态水印、极简数据叠加等酷炫卡片）。
    *   **社群：** 强依赖微信群能力，支持一键在群内发起“周末约跑班”，并在群聊卡片内实时更新大家的总公里数。

## 5. 结论：V1.0 MVP 功能清单建议

根据调研，我们剔除了需要重度运营和硬件接入的功能，保留最核心体验：

| 优先级 | 功能模块 | 描述 |
| :--- | :--- | :--- |
| **P0 (必做)** | 核心记录 | 微信实时定位（后台保活权限）、经纬度轨迹绘制、基础数据（时间、距离、配速/速度估算）。 |
| **P0 (必做)** | 数据总结与卡片 | 运动结束后的数据总览图、支持更换底图照片的分享海报生成（Canvas技术）。 |
| **P1 (重点)** | 微信分享闭环 | 生成适合朋友圈的高清长图、生成适合微信群分享的小程序动态卡片。 |
| **P1 (重点)** | 历史记录列表 | 本地或云端保存用户的历史运动记录，提供按月统计。 |
| **P2 (未来)** | 语音播报 | 每公里语音播报（利用微信TTS或音频文件）。 |
| **P2 (未来)** | 路线路书导入 | 导入别人骑行过的GPX路线并在地图上显示作为导航参考。 |
