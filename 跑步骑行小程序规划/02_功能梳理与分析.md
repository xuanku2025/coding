# 02 功能梳理与分析

基于前期的竞品调研和产品定位（轻量、纯粹、易分享的微信工具），我们将跑步骑行小程序的功能进行全面梳理，并按照 MVP（最小可行性产品）的原则划分为不同的优先级。

## 1. 核心业务流程

用户在使用本小程序时的典型路径如下：

**打开小程序 -> 选择运动类型（跑步/骑行） -> 确认 GPS 信号 -> 点击“开始” -> 运动中（实时查看数据、轨迹、支持暂停/继续） -> 长按“结束” -> 结算页展示路线与数据 -> （可选）生成带照片的分享海报 -> 分享到微信朋友圈或会话 -> 返回首页/历史记录。**

---

## 2. 模块功能详细拆解 (V1.0 MVP)

我们将 V1.0 版本的核心功能划分为四个主要模块：**首页控制台**、**运动记录(核心)**、**数据总结与分享**、**个人中心与历史**。

### 模块一：首页控制台 (Home)

此模块是用户打开小程序的首屏，强调快速响应和“即刻出发”的体验。

| 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- |
| **全屏地图底图** | 首页背景为当前实时位置的地图，中心点展示用户当前位置的定位图标。 | P0 |
| **GPS 信号强度提示** | 顶部或显著位置显示当前 GPS 信号状态（弱/良/优），提示用户是否适合开始运动以保证轨迹精度。 | P0 |
| **运动类型切换** | 提供明显（大尺寸/居中）按钮，快速切换**跑步**和**骑行**模式，改变对应的核心数据计算逻辑和展示样式。 | P0 |
| **一键“开始”按钮** | 底部巨大的圆形按钮，点击即可开启倒计时（3,2,1）并进入运动记录状态。 | P0 |
| **权限引导阻断** | 当检测到未授权“使用时定位”或“设置-位置信息-始终/使用小程序期间”时，弹出友好的弹窗引导至设置页授权。 | P0 |

### 模块二：运动记录 (In-Workout) - **核心难点**

此模块涵盖用户在运动全过程中的体验，包括数据的实时获取、计算和展示。此部分的稳定性决定了产品的生死。

| 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- |
| **后台定位保活机制** | 核心技术：在微信切后台、手机息屏时（需配置 `requiredBackgroundLocation`），持续获取并记录经纬度。处理轨迹漂移和中断重连。 | P0（硬核） |
| **核心数据实时计算** |
| - *运动时间* | 从点击开始到当前经过的有效时长（排除暂停时间），格式：`MM:SS` 或 `HH:MM:SS`。 | P0 |
| - *运动总距离* | 基于经纬度序列，剔除异常漂移点后，累加两点间距离得出，单位：公里（保留两位小数，如 `5.20 km`）。 | P0 |
| - *当前配速 (跑)* | 取最近若干定位点的速度换算后的平均值，格式：`分'秒"/公里`（如 `5'30"`）。 | P0 |
| - *当前速度 (骑)* | 取最近定位点的瞬时速度，单位：`km/h`。 | P0 |
| **轨迹实时绘制** | 在地图组件上（如 `map.addPolyline`），用醒目的颜色（如渐变色）将走过的点连成平滑的线。 | P0 |
| **防误触控制面板** | 提供大面积的**“暂停/继续”**按钮，必须**长按或滑动解锁**才能点击**“结束”**按钮，防止误操作中断记录。 | P0 |

*进阶功能（V1.x迭代）：*
| 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- |
| *整公里振动/语音提示* | 每完成 1km，调用手机振动（低门槛）或微信 TTS（高门槛）播报上一公里的配速。 | P1 |
| *常亮锁屏面板* | 利用小程序 `wx.setKeepScreenOn` 保持运动过程中屏幕常亮（用户可选开启，默认建议开启）。 | P1 |

### 模块三：数据总结与分享 (Summary & Share) - **增长引擎**

用户完成运动后的“爽感”来源，通过极简的数据展示和高颜值的海报生成刺激用户分享到微信生态。

| 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- |
| **运动详情总览页** | 包含静态地图截屏（展示全程轨迹线）、核心数据大字报（总距离、总时长、平均配速/速度、卡路里估算）。 | P0 |
| **异常数据处理逻辑** | 如果距离极短（如小于0.1km）或时间极短，提示“距离过短是否舍弃”，避免产生垃圾数据。 | P0 |
| **一键微信分享卡片** | 点击按钮直接调用 `wx.showShareMenu` 和 `onShareAppMessage`，生成带有小程序特定页面路径和总公里数的群聊分享卡片。 | P0 |
| **长图海报生成 (Canvas)** | **核心卖点**：将用户的地图轨迹（可切换为静态图或卫星图底图）与精美的文字排版（水印风格）通过 Canvas 合成为一张高清图片。 | P1 (极力推荐 V1 上) |
| **照片底图叠加 (海报进阶)** | 允许用户调起相册，选择一张跑步时拍的照片作为海报全屏背景，将半透明的轨迹图和数据文字叠加其上生成。 | P1 |

*进阶功能（V1.x迭代）：*
| 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- |
| *配速/海拔图表绘制* | 使用可视化库（如 Echarts-for-weixin 或 F2）展示逐公里的柱状图（配速）和折线图（海拔起伏）。 | P2 |

### 模块四：个人中心与历史记录 (Profile & History)

用于沉淀用户数据，建立长期使用的习惯，增强用户粘性。

| 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- |
| **个人概览聚合** | 展示用户的头像/昵称，以及（跑步/骑行分开的）历史总里程、累计运动次数、最长单次记录等勋章数据。 | P0 |
| **历史运动列表** | 按时间倒序（按月分组）列出历史运动记录块，每条记录包含：日期、运动类型图标、距离、时长、平均配速。 | P0 |
| **云端数据同步** | 每次运动结束后，异步将关键数据点（摘要+简化版轨迹点）上传至云数据库（如微信云开发），确保换手机、卸载重装数据不丢失。 | P1 |
| **历史记录追溯** | 点击列表项可重新进入该次运动的详情总结页（回顾当时的轨迹和数据，可再次生成海报）。 | P1 |

*进阶功能（V1.x迭代）：*
| 功能点 | 描述 | 优先级 |
| :--- | :--- | :--- |
| *月度/年度报告* | 提供更宏观的数据图表分析报告。 | P2 |
| *删除与管理记录* | 允许用户手动删除不满意的历史脏数据。 | P2 |

## 3. 功能优先级总结与版本规划

综上分析，我们为 MVP (V1.0) 版本制定了必须上线的“生死线”功能，这保证了产品能够走通核心闭环：

**MVP 版本 (V1.0) 必须包含：**
1.  **首页控制台：** 实时地图展示，运动类型切换，一键开始功能。
2.  **运动记录：** 必须申请并跑通**微信后台定位权限**，实现基本的轨迹绘制和距离、时间、配速计算，包含防误触的暂停和长按结束逻辑。
3.  **数据总结与群分享：** 结束后的核心数据页，以及支持发送给微信好友/群聊的小程序默认分享卡片。
4.  **云端存储与简易列表：** 利用最简捷的云服务保存运动摘要，并在“我的”页面列出历史记录。

**后续迭代 (V1.1 - 1.2) 的重点：**
1.  **病毒式传播组件：** 支持上传照片合成的“水印轨迹海报”长图生成。
2.  **体验优化：** 轨迹漂移算法优化、息屏常亮控制、更细腻的图表分析（配速/海拔曲线）。
