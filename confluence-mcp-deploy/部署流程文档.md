# Confluence MCP 连接 Figma Make — 完整部署流程

本文档沉淀将内网 Confluence 通过 MCP 连接至 Figma Make 的完整部署流程，包括环境要求、参数配置、操作步骤及常见问题。

---

## 一、架构说明

```
┌─────────────────┐     HTTPS      ┌──────────────────┐     HTTP      ┌─────────────────────┐
│  Figma Make     │ ──────────────► │  内网穿透        │ ───────────► │  mcp-atlassian     │
│  (云端)         │                 │  (ngrok/         │              │  (Docker 容器)      │
│                 │                 │   cloudflared)   │              │                     │
└─────────────────┘                 └──────────────────┘              └──────────┬──────────┘
                                                                                  │
                                                                                  │ HTTP REST API
                                                                                  ▼
                                                                         ┌─────────────────────┐
                                                                         │  Confluence         │
                                                                         │  (内网)              │
                                                                         │  confluence.zstack.io│
                                                                         └─────────────────────┘
```

> **要点**：Figma Make 要求 MCP Server URL 为 **HTTPS 且公网可访问**，Confluence 部署在内网，因此需要「MCP Server + 内网穿透」作为桥梁。

## 二、环境与资源要求

### 2.1 必需资源

| 资源类型 | 要求 | 说明 |
|----------|------|------|
| **虚拟机** | 1 台 | 需能访问 Confluence 内网地址，且可访问公网（用于内网穿透） |
| **Docker** | 已安装 | 用于运行 mcp-atlassian 容器 |
| **Confluence** | Server/Data Center v6.0+ | 本文以 6.15.9 为例 |
| **Confluence 账号** | 用户名 + 密码 | 用于 MCP 调用 Confluence REST API |

### 2.2 端口要求

| 端口 | 用途 | 说明 |
|------|------|------|
| **9090** | MCP 默认端口 | 若被占用（如 clash），可改用 9092 等 |
| **9092** | 备用端口 | 本文实际使用，因 9090 被 clash 占用 |

### 2.3 网络要求

- 虚拟机可访问：`http://confluence.zstack.io`
- 虚拟机可访问公网（用于 cloudflared/ngrok 连接 Cloudflare/ngrok 服务）

## 三、参数配置

### 3.1 环境变量（.env）

| 参数名 | 必填 | 说明 | 示例值 |
|--------|------|------|--------|
| `CONFLUENCE_URL` | 是 | Confluence 地址，不要末尾斜杠 | `http://confluence.zstack.io` |
| `CONFLUENCE_USERNAME` | 是 | 登录用户名 | `enyou.sun` |
| `CONFLUENCE_API_TOKEN` | 是* | 旧版 Confluence 填**登录密码**；有 Token 时填 Token | `你的密码` |
| `CONFLUENCE_PERSONAL_TOKEN` | 否 | Confluence 7.14+ 的 PAT，与 API_TOKEN 二选一 | — |
| `CONFLUENCE_SSL_VERIFY` | 否 | 内网 HTTP 时建议 `false` | `false` |
| `MCP_PORT` | 否 | MCP 容器内端口，默认 9090 | `9090` |

> *Confluence 6.x / 许可证过期时，官方文档建议将**登录密码**填在 `CONFLUENCE_API_TOKEN` 中。

### 3.1.1 Jira 认证（如需同时启用 Jira）

若需要同时连接 Jira，可在 `.env` 中补充 Jira 配置。你们环境若使用「用户名+密码」，可先按 Basic Auth 写法：

```env
JIRA_URL=http://jira.your-company.com
JIRA_USERNAME=你的用户名
JIRA_API_TOKEN=你的登录密码
```

> 兼容性提醒：Jira Cloud 通常不支持账号密码直连，需要 API Token；Jira Server/Data Center 是否允许 Basic Auth 取决于实例安全策略。

### 3.2 Figma Make Connector 参数

| 参数 | 说明 | 示例 |
|------|------|------|
| **MCP server URL** | 内网穿透后的 HTTPS 地址 + 端点 | `https://xxx.trycloudflare.com/mcp` |
| **端点路径** | **推荐**：streamable-http 用 `/mcp`；（可选）SSE 用 `/sse` | `/mcp` |

## 四、部署步骤

### 步骤 1：在虚拟机上创建部署目录

```bash
mkdir -p ~/confluence-mcp-deploy
cd ~/confluence-mcp-deploy
```

### 步骤 2：创建 .env 配置文件

```bash
cat > .env << 'EOF'
CONFLUENCE_URL=http://confluence.zstack.io
CONFLUENCE_USERNAME=enyou.sun
CONFLUENCE_API_TOKEN=你的Confluence登录密码
CONFLUENCE_SSL_VERIFY=false
MCP_PORT=9090
EOF
```

**重要**：用 `vi .env` 或 `nano .env` 编辑，将 `你的Confluence登录密码` 替换为真实密码。  
**注意**：若用 heredoc 创建，确保文件末尾没有多余 `ENVEOF` 等标记，且首行是 `CONFLUENCE_URL`（不要漏掉 `C`）。

### 步骤 3：检查端口占用

```bash
ss -tlnp | grep 9090
```

若 9090 已被占用（如 clash），后续使用 **9092** 映射。

### 步骤 4：启动 mcp-atlassian 容器

```bash
# 若存在旧容器，先清理
docker rm -f mcp-atlassian 2>/dev/null || true

# 使用 9092 映射（避开 9090 占用）
docker run -d \
  --name mcp-atlassian \
  --env-file ~/confluence-mcp-deploy/.env \
  -e MCP_VERBOSE=true \
  -p 9092:9090 \
  ghcr.io/sooperset/mcp-atlassian:latest \
  --transport streamable-http --port 9090
```

### 步骤 5：验证 MCP 服务（推荐 /mcp）

```bash
curl -v -N -H "Accept: text/event-stream" http://localhost:9092/mcp
```

预期：返回 `HTTP/1.1 200 OK`，`content-type: text/event-stream`（若不加该 Accept 头，可能返回 406）。

### 步骤 6：安装并启动内网穿透（二选一）

#### 方案 A：Cloudflare Tunnel

```bash
# 安装（若未安装）
wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 -O /usr/local/bin/cloudflared
chmod +x /usr/local/bin/cloudflared

# 启动（端口与 Docker 映射一致）
cloudflared tunnel --url http://localhost:9092
```

记下输出的 `https://xxx.trycloudflare.com` 地址。

#### 方案 B：ngrok

```bash
# 解压并运行
cd ~
tar -xzf ngrok-v3-stable-linux-amd64.tgz
./ngrok http 9092
```

记下输出的 `https://xxx.ngrok-free.app` 地址。

### 步骤 7：在 Figma Make 中配置 Connector

1. 打开 Figma Make → **Connectors** → **Create** → **Custom Connector**
2. **Step 1 of 2**：填写 Name、Tagline、Description（如 Name: `confluence`）
3. **Step 2 of 2**：
   - **MCP server URL**：`https://你的公网地址/mcp`  
     例如：`https://interact-invoice-statements-restaurant.trycloudflare.com/mcp`
   - Advanced settings 可留空（认证已在 MCP 端配置）
4. 保存并测试连接

### 步骤 8：测试用例

在 Figma Make 聊天框中输入：

```
搜索 Confluence 中关于「设计」或「UI」的页面
```

或：

```
列出 Confluence 中所有的空间（spaces）
```

若返回 Confluence 数据，则部署成功。

## 五、常见问题与处理

### 5.1 端口被占用（bind: address already in use）

**现象**：`docker run` 报错 `listen tcp4 0.0.0.0:9090: bind: address already in use`

**处理**：
```bash
ss -tlnp | grep 9090   # 查看占用进程
# 改用其他端口映射，如 -p 9092:9090
```

### 5.2 curl 返回 Connection reset by peer

**可能原因**：curl 未加 `-N` 或 `Accept: text/event-stream`，或容器刚启动未就绪。

**处理**：
```bash
curl -v -N -H "Accept: text/event-stream" http://localhost:9092/sse
```

### 5.3 .env 中 CONFLUENCE_URL 错误

**现象**：首行显示为 `ONFLUENCE_URL`（少 `C`），或末尾有 `ENVEOF`。

**处理**：用 `vi .env` 修正，确保：
- 第一行：`CONFLUENCE_URL=http://confluence.zstack.io`
- 最后一行：`MCP_PORT=9090`，无多余内容

### 5.4 Figma 一直显示 Connecting...

**可能原因**：
- MCP URL 未加 `/sse` 后缀
- trycloudflare 对自动化客户端有拦截

**处理**：
- 确认 URL 格式：`https://xxx.trycloudflare.com/sse`
- 可尝试 streamable-http 的 `/mcp` 端点
- 或改用 ngrok 替代 cloudflared

### 5.5 401 Unauthorized

**处理**：检查 `CONFLUENCE_USERNAME` 与 `CONFLUENCE_API_TOKEN`（密码）是否正确，Confluence 是否允许该账号登录。

## 七、使用注意事项与运维

### 7.1 使用注意事项

| 注意事项 | 说明 |
|----------|------|
| **内网穿透需常驻** | cloudflared / ngrok 需**持续运行**，关闭终端或进程后公网地址失效，Figma 将无法连接 |
| **公网地址会变化** | trycloudflare / ngrok 免费版每次重启会生成**新地址**，需在 Figma 中更新 MCP server URL |
| **密码安全** | `.env` 含 Confluence 密码，**不要提交到 Git**，不要分享给他人 |
| **生产环境** | 长期使用建议：自有域名 + Nginx 反向代理 + 固定 HTTPS，避免地址频繁变更 |
| **Confluence 密码变更** | 若修改 Confluence 登录密码，需同步更新 `.env` 并**重启 mcp-atlassian 容器** |

### 7.2 部署在虚拟机上，后续如何启用

#### 方式一：一键脚本（推荐）

将 `start-and-get-url.sh` 放到虚拟机的 `~/confluence-mcp-deploy/` 目录下（可从本仓库复制或 scp 上传），虚拟机重启后执行：

```bash
cd ~/confluence-mcp-deploy
chmod +x start-and-get-url.sh
./start-and-get-url.sh
```

脚本会自动：
1. 启动 mcp-atlassian 容器（若未运行）
2. 启动 cloudflared 隧道（后台常驻）
3. 解析并输出 **MCP server URL**，直接复制到 Figma 配置即可

输出示例：
```
========================================
  Confluence MCP 已启动
========================================

  请在 Figma Make 的 Custom Connector 中配置：

  MCP server URL:
  https://xxx.trycloudflare.com/mcp
```

**可选环境变量**（在脚本前设置）：
- `TUNNEL_TYPE=ngrok`：使用 ngrok 替代 cloudflared
- `MCP_HOST_PORT=9092`：若端口不同可修改

#### 方式二：手动启动

若服务已停止，按以下顺序启动：

```bash
# 1. 启动 mcp-atlassian 容器
docker start mcp-atlassian

# 2. 启动内网穿透（需保持终端运行）
cloudflared tunnel --url http://localhost:9092
# 或：./ngrok http 9092
```

**验证**：
```bash
docker ps | grep mcp-atlassian   # 确认容器在运行
curl -s -o /dev/null -w "%{http_code}" -H "Accept: text/event-stream" http://localhost:9092/mcp   # 应返回 200
```

### 7.3 虚拟机重启后的处理

| 组件 | 虚拟机重启后 | 是否需要手动操作 |
|------|--------------|------------------|
| **Docker 服务** | 随系统自动启动 | 否（Docker 通常配置为开机自启） |
| **mcp-atlassian 容器** | **默认不会自动启动** | **是**，需手动 `docker start mcp-atlassian` 或配置自启 |
| **cloudflared / ngrok** | **不会自动启动** | **是**，需手动启动或配置为 systemd 服务 |

### 7.4 配置开机自启（推荐）

#### mcp-atlassian 容器自启（推荐）

重新创建容器时加上 `--restart unless-stopped`：

```bash
docker rm -f mcp-atlassian 2>/dev/null || true

docker run -d \
  --name mcp-atlassian \
  --restart unless-stopped \
  --env-file ~/confluence-mcp-deploy/.env \
  -e MCP_VERBOSE=true \
  -p 9092:9090 \
  ghcr.io/sooperset/mcp-atlassian:latest \
  --transport streamable-http --port 9090
```

> 虚拟机重启后，Docker 会**自动拉起**该容器。

#### cloudflared 自启（systemd，推荐）

推荐直接使用本仓库的安装脚本一键创建并启用 systemd 服务（会自动把最新公网 URL 写入文件）：

```bash
cd ~/confluence-mcp-deploy
chmod +x install-autostart-systemd.sh
sudo ./install-autostart-systemd.sh

# 查看状态
systemctl status cloudflared-mcp

# 查看当前可复制到 Figma 的 URL（含 /mcp）
cat ~/confluence-mcp-deploy/MCP_URL.txt
```

> **注意**：cloudflared 每次重启会生成**新地址**，需在 Figma 中更新 MCP URL。若需固定地址，请使用 Cloudflare 命名隧道或自有域名。

### 7.5 日常运维命令速查

| 操作 | 命令 |
|------|------|
| **一键启动并获取 Figma URL** | `~/confluence-mcp-deploy/start-and-get-url.sh`（会写入 `MCP_URL.txt`） |
| 查看容器状态 | `docker ps \| grep mcp-atlassian` |
| 查看容器日志 | `docker logs mcp-atlassian --tail 50` |
| 重启容器 | `docker restart mcp-atlassian` |
| 停止容器 | `docker stop mcp-atlassian` |
| 启动容器 | `docker start mcp-atlassian` |
| 停止 tunnel | `pkill -f 'cloudflared tunnel'` 或 `pkill -f ngrok` |
| 查看 tunnel 日志 | `tail -f /tmp/confluence-mcp-tunnel.log` |
| 查看 cloudflared 状态 | `systemctl status cloudflared-mcp`（若已配置 systemd） |
