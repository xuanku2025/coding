# Confluence MCP 连接 Figma Make — 可行方案（按当前条件）

基于 [MCP Atlassian 官方文档](https://mcp-atlassian.soomiles.com/docs)，结合你当前环境整理的可执行方案。

| 项目 | 你的情况 | 说明 |
|------|----------|------|
| Confluence | `http://confluence.zstack.io/`，**6.15.9** | 官方兼容性：**Server/Data Center v6.0+ 支持** ✅ |

## 认证方案（二选一，优先用 ①）

官方 [Troubleshooting](https://mcp-atlassian.soomiles.com/docs/troubleshooting) 写明：

> **Authentication Failures (Server/Data Center)**  
> For older Confluence servers, try basic auth with `CONFLUENCE_USERNAME` and `CONFLUENCE_API_TOKEN` **(where token is your password)**.

即：**旧版 Confluence 可用「用户名 + 密码」**，密码填在 `CONFLUENCE_API_TOKEN` 里。

| 方式 | 适用情况 | 环境变量 |
|------|----------|----------|
| **① 用户名 + 密码** | 许可证过期、API Token 不可用时 | `CONFLUENCE_USERNAME` + `CONFLUENCE_API_TOKEN=你的登录密码` |
| **② API Token / PAT** | 若页面能创建且可用 | `CONFLUENCE_USERNAME` + `CONFLUENCE_PERSONAL_TOKEN=Token` 或 `CONFLUENCE_API_TOKEN=Token` |

建议：**先按 ① 用用户名+密码**，能跑通再考虑 Token。

## 步骤 1：准备 .env（用户名+密码方式）

在 `confluence-mcp-deploy` 目录下：

```bash
cd confluence-mcp-deploy
cp .env.example .env
```

编辑 `.env`，**按用户名+密码方式**填写（不要提交到 Git）：

```env
# Confluence 内网地址（不要末尾斜杠）
CONFLUENCE_URL=http://confluence.zstack.io

# 登录用户名
CONFLUENCE_USERNAME=enyou.sun

# 旧版 Confluence：这里填你的 Confluence 登录密码（官方文档：token 即 password）
CONFLUENCE_API_TOKEN=你的登录密码

# 内网为 HTTP 时可关闭 SSL 校验（可选）
CONFLUENCE_SSL_VERIFY=false

# MCP 端口（与下面 docker 一致即可）
MCP_PORT=9090
```

若你之后改用「API Token」且可用，可改为：

```env
CONFLUENCE_PERSONAL_TOKEN=你的API_Token
# 或 CONFLUENCE_API_TOKEN=你的API_Token
```

---

## 步骤 2：在虚拟机上用 Docker 跑 MCP（SSE）

选一台**能访问** `http://confluence.zstack.io` **的虚拟机**，执行：

```bash
# 确保 .env 已在该目录
docker run -d \
  --name mcp-atlassian \
  --env-file .env \
  -p 9090:9090 \
  ghcr.io/sooperset/mcp-atlassian:latest \
  --transport sse \
  --port 9090
```

验证：

```bash
curl -v http://localhost:9090/sse
```

有 SSE 流或 200 即表示 MCP 已启动。Figma Make 需要的是 **HTTPS + 公网**，所以还要做步骤 3。

---

## 步骤 3：把 MCP 暴露为公网 HTTPS（Figma 要求）

Figma Make 的 Custom Connector 要求 MCP Server URL 为 **`https://` 且公网可访问**。任选一种方式。

### 方案 A：ngrok（最快）

在同一台虚拟机上：

```bash
ngrok http 9090
```

记下给出的 HTTPS 地址，例如 `https://abc123.ngrok-free.app`。  
Figma 里填：**`https://abc123.ngrok-free.app/sse`**（必须带 `/sse`）。

### 方案 B：Cloudflare Tunnel

```bash
cloudflared tunnel --url http://localhost:9090
```

用生成的 `https://xxx.trycloudflare.com`，Figma 填：**`https://xxx.trycloudflare.com/sse`**。

### 方案 C：自有域名 + Nginx

在公网机器上用 Nginx 把 `https://你的域名` 反代到 `http://内网虚拟机IP:9090`，并配置 SSL。  
Figma 填：**`https://你的域名/sse`**。

---

## 步骤 4：在 Figma Make 里配置 Connector

1. 打开 Figma Make → Custom Connector。
2. **Step 1 of 2**：Name / Tagline / Description 按需填写（如 Name: `confluence`）。
3. **Step 2 of 2**：
   - **MCP server URL**：填你在步骤 3 得到的 **HTTPS 地址 + `/sse`**（如 `https://abc123.ngrok-free.app/sse`）。
   - 认证已在 MCP 端用 `.env` 配好，Figma 的 Additional headers 可留空。
4. 保存并测试连接。

---

## 调试（若 401/403）

官方建议开启详细日志：

```bash
# 运行时加环境变量
docker run -d \
  --name mcp-atlassian \
  --env-file .env \
  -e MCP_VERBOSE=true \
  -e MCP_VERY_VERBOSE=true \
  -p 9090:9090 \
  ghcr.io/sooperset/mcp-atlassian:latest \
  --transport sse --port 9090
```

- **401**：检查 `CONFLUENCE_USERNAME` 与 `CONFLUENCE_API_TOKEN`（密码）是否正确、Confluence 是否允许该账号登录。
- **SSL 报错**：已用 `CONFLUENCE_SSL_VERIFY=false` 时，确认 `.env` 被 `--env-file` 正确加载。
