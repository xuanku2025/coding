# UI 视觉走查工具 PRD + 技术方案

---

## 一、产品概述

| 项目 | 内容 |
|------|------|
| 产品名称 | UI Visual Inspector |
| 产品定位 | 设计师通过 Figma 链接获取设计稿，捕获前端页面截图，视觉对比后一键提交 Jira Bug |
| 目标用户 | UI 设计师、视觉 QA、测试工程师 |

## 三、功能列表

### 3.1 设计稿输入

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Figma 链接输入 | 输入 Figma 链接，自动解析 fileKey 和 nodeId | P0 |
| Figma MCP 获取截图 | 调用 get_design_context 获取设计稿截图 | P0 |
| 手动上传图片 | 支持拖拽上传 PNG/JPG 作为备选方案 | P1 |

### 3.2 前端页面截图

| 功能 | 描述 | 优先级 |
|------|------|--------|
| URL 输入框 | 输入需要走查的页面 URL | P0 |
| 视口尺寸选择 | Desktop (1920×1080) / Mobile (375×667) / Tablet (768×1024) | P1 |
| Puppeteer 截图 | 自动导航到 URL 并截图 | P0 |
| 指定元素截图 | 可选择特定 DOM 元素进行截图 | P2 |

### 3.3 视觉对比

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 滑块对比 | 左右拖动滑块对比设计稿与实装页面 | P0 |
| 叠加对比 | 半透明叠加显示差异 | P1 |
| 差异高亮 | 自动标记像素差异区域 | P1 |
| 缩放 / 平移 | 支持缩放和平移操作 | P1 |

### 3.4 差异标注

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 点击标记 | 在对比图上点击标记差异点 | P0 |
| 问题类型选择 | 颜色 / 字体 / 间距 / 尺寸 / 其他 | P0 |
| 问题描述 | 填写详细的问题描述 | P0 |
| 截图截取 | 自动截取差异区域作为证据 | P0 |
| 问题列表 | 显示所有已标注的问题，支持删除 | P0 |

### 3.5 Jira 集成

| 功能 | 描述 | 优先级 |
|------|------|--------|
| Jira 配置 | 填写实例地址、邮箱、API Token | P0 |
| 项目选择 | 选择目标 Jira 项目 | P0 |
| Issue Type | 选择 Bug | P0 |
| 自动创建 Issue | 提交时自动创建 Jira Bug | P0 |
| 标题模板 | `[UI走查] {页面名} - {问题描述}` | P0 |
| 描述模板 | 包含问题描述 + 截图 | P0 |
| 多个 Bug 批量提交 | 一次提交多条 Bug | P1 |

### 3.6 其他功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 历史记录 | 保存走查历史 | P2 |
| 报告导出 | 导出 PDF 走查报告 | P2 |

### 5.3 核心服务模块

#### FigmaService
```typescript
// 使用 Figma MCP 获取设计稿截图
import { mcp__figma__get_design_context } from './figma-mcp';

async function getDesignScreenshot(figmaUrl: string) {
  const { fileKey, nodeId } = parseFigmaUrl(figmaUrl);
  const result = await mcp__figma__get_design_context({
    fileKey,
    nodeId,
    excludeScreenshot: false
  });
  return result.screenshot;
}
```

#### CaptureService
```typescript
// Puppeteer 截图
import puppeteer from 'puppeteer';

async function capturePage(url: string, viewport: Viewport) {
  const browser = await puppeteer.launch({ headless: true });
  const page = await browser.newPage();
  await page.setViewport(viewport);
  await page.goto(url, { waitUntil: 'networkidle0' });
  const screenshot = await page.screenshot({ encoding: 'base64' });
  await browser.close();
  return screenshot;
}
```

#### JiraService
```typescript
// Jira REST API 提交 Bug
import axios from 'axios';

class JiraService {
  private client: axios;

  constructor(baseUrl: string, email: string, apiToken: string) {
    this.client = axios.create({
      baseURL: `${baseUrl}/rest/api/3`,
      auth: { username: email, password: apiToken }
    });
  }

  async createBug(projectKey: string, summary: string, description: string, screenshot: string) {
    // 上传截图附件
    const attachment = await this.uploadAttachment(screenshot);

    // 创建 Issue
    return this.client.post('/issue', {
      fields: {
        project: { key: projectKey },
        summary,
        description: {
          type: 'doc',
          version: 1,
          content: [{ type: 'paragraph', content: [{ type: 'text', text: description }] }]
        },
        issuetype: { name: 'Bug' },
        attachment: [attachment]
      }
    });
  }
}
```

## 六、Jira 提交规范

### Issue 标题格式
```
[UI走查] {页面名称} - {问题简述}
```

示例：
```
[UI走查] 首页 - 按钮颜色不一致
[UI走查] 个人中心 - 标题字体大小偏小
```

### Issue 描述模板
```markdown
## 问题类型
颜色 / 字体 / 间距 / 尺寸 / 其他

## 问题描述
{详细描述}

## 对比截图
{自动附加截图}

## 发现时间
{自动记录}
```

### 自定义字段（可选）
- 标签：`ui-inspector`
- 优先级：根据问题类型自动选择

## 九、风险与限制

| 风险 | 影响 | 解决方案 |
|------|------|----------|
| Figma 私有文件 | 无法获取私有文件截图 | 需要 Figma 插件配合或导出图片 |
| 页面需要登录 | 无法截图 | 支持手动上传登录后的页面截图 |
| 跨域问题 | 页面无法截图 | Puppeteer 可解决大部分情况 |
| 动态内容 | 截图时机不确定 | 提供延时截图选项 |
